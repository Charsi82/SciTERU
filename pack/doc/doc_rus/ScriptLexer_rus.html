Скриптовый лексер для SciTE

Создание лексера на Lua

Лексер может быть написан в виде скрипта на языке Lua вместо C++. Это немного проще и позволяет разрабатывать лексеры без использования компилятора C++.

Лексер скрипта подключается путём указания имени файла лексера, начинающегося с "script_". Затем, используя это имя, можно назначать стили и другие свойства. Например,

lexer.*.zog=script_zog
style.script_zog.0=fore:#7f007f,bold
style.script_zog.1=fore:#000000
style.script_zog.2=fore:#000080,bold
style.script_zog.3=fore:#008000,font:Georgia,italics,size:9

Тогда лексер реализуется на Lua подобно этому:

-- -*- coding: utf-8 -*-

function OnStyle(styler)
        local S_DEFAULT = 0
        local S_IDENTIFIER = 1
        local S_KEYWORD = 2
        local S_UNICODECOMMENT = 3
        local identifierCharacters = "abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ"

        styler:StartStyling(styler.startPos, styler.lengthDoc, styler.initStyle)
        while styler:More() do

                -- Exit state if needed
                if styler:State() == S_IDENTIFIER then
                        if not identifierCharacters:find(styler:Current(), 1, true) then
                                local identifier = styler:Token()
                                if identifier == "if" or identifier == "end" then
                                        styler:ChangeState(S_KEYWORD)
                                end
                                styler:SetState(S_DEFAULT)
                        end
                elseif styler:State() == S_UNICODECOMMENT then
                        if styler:Match("»") then
                                styler:ForwardSetState(S_DEFAULT)
                        end
                end

                -- Enter state if needed
                if styler:State() == S_DEFAULT then
                        if styler:Match("«") then
                                styler:SetState(S_UNICODECOMMENT)
                        elseif identifierCharacters:find(styler:Current(), 1, true) then
                                styler:SetState(S_IDENTIFIER)
                        end
                end

                styler:Forward()
        end
        styler:EndStyling()
end

Результат выглядит так:

proc clip(int a)
« Clip into the positive zone »
if (a > 0) a
0
end

Структура кода

Цикл документа

Лексер проходит по указанной части документа, присваивая каждому символу свой стиль.

styler:StartStyling(styler.startPos, styler.lengthDoc, styler.initStyle)
while styler:More() do
        -- Code that examines the text and sets lexical states
        styler:Forward()
end
styler:EndStyling()

Существует множество различных способов структурирования кода, который анализирует текст и устанавливает лексические состояния. Структура, оказавшаяся полезной в лексерах C++, — это написание двух блоков кода, как показано в примере. Первый блок проверяет, должно ли текущее состояние завершиться, и если да, то устанавливает состояние в значение по умолчанию 0. Второй блок отвечает за определение того, следует ли перейти в новое состояние из состояния по умолчанию. Такая структура позволяет обрабатывать все как переключение из состояния по умолчанию или обратно и избежать необходимости рассматривать множество комбинаций состояний.

Кодировки

Стайлер перебирает целые символы, а не байты. Таким образом, если документ закодирован в UTF-8, styler:Current() может представлять собой многобайтовую строку. Если же скрипт также закодирован в UTF-8, то легко проверить его на соответствие символам Юникода с помощью такого кода, как

if styler:Current() == "«" then

Если используется кодировка, например Latin-1, и сам шрифт также закодирован в той же кодировке, то литералы можно использовать, как указано выше.

Если язык может быть закодирован различными способами, то может потребоваться более сложный код, а также код, специфичный для данной кодировки.

Предварительные проверки

Иногда лексеру необходимо увидеть некоторую информацию в начале файла, например, объявление изменяет синтаксис или конкретная форма кавычек. Кавычки в начале строки должны совпадать с кавычками в конце. Поскольку стандартный цикл идет только вперед от начальной позиции, необходимо использовать другие вызовы, такие как CharAt и StyleAt. Они используют позиции байтов и не рассматривают многобайтовые символы как единые сущности.

Производительность

Приведённый выше лексический анализатор способен обрабатывать приблизительно 90 000 запросов в секунду на процессоре Athlon 64 с частотой 2,4 ГГц. В большинстве ситуаций это будет ощущаться совершенно плавно.

Более сложные лексеры будут работать медленнее. Если лексер настолько медленный, что приложение перестаёт отвечать, то лексер может разделить каждый запрос. Он может сделать это, выбрав диапазон целых строк и используя этот диапазон в качестве аргументов функции StartStyling. Это позволяет обрабатывать нажатия клавиш и движения мыши пользователя. Лексер будет автоматически вызван снова для лексического анализа большей части документа.


Программый интерфейс

Пример лексера построчной обработки.

Этот пример предназначен для построчно-ориентированного языка, который иногда используется для конфигурационных файлов. В нём используются низкоуровневые прямые вызовы вместо вызовов StartStyling/More/Forward/EndStyling.

-- Лексер, ориентированный на строки — стилизует строку в соответствии с первым символом.
function OnStyle(styler)
        local lineStart = editor:LineFromPosition(styler.startPos)
        local lineEnd = editor:LineFromPosition(styler.startPos + styler.lengthDoc)
        editor:StartStyling(styler.startPos, 31)
        for line=lineStart, lineEnd, 1 do
                local lengthLine = editor:PositionFromLine(line + 1) - editor:PositionFromLine(line)
                local lineText = editor:GetLine(line)
                local first = string.sub(lineText, 1, 1)
                local style = 0
                if first == "+" then
                        style = 1
                elseif first == " " or first == "\t" then
                        style = 2
                end
                editor:SetStyling(lengthLine, style)
        end
end
